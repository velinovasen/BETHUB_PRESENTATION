Walk-Forward Validation Methodology

🔑 Key Innovation

Walk-forward validation achieves 1,200% ROI with 42.15% accuracy in production

Highly selective prediction rate (11.32%) ensures only high-confidence predictions are made, validated across 3,206 games in top performing markets.

Walk-Forward Validation Process

Core Principle: For each prediction at time T, only use data from times < T for threshold optimization. This prevents look-ahead bias where the model "knows" future patterns.
sequenceDiagram participant Historical as Historical Data participant Train as Training Window participant Calibrate as Calibration Window participant Predict as Prediction Point Note over Historical,Predict: Time flows left to right → Historical->>Train: Use past data for training Train->>Calibrate: 150-365 day window Calibrate->>Calibrate: Evaluate 101 threshold candidates Calibrate->>Predict: Apply optimal threshold Note over Calibrate: Only use data BEFORE prediction time Note over Predict: Never use future data Predict->>Calibrate: Roll forward to next prediction Calibrate->>Calibrate: Recalibrate thresholds

Walk-Forward vs Standard Backtesting

Problem with Standard Backtesting: It uses ALL data (including future) to find the "optimal" threshold, then applies it to historical predictions. This creates look-ahead bias.
gantt title Threshold Optimization: Standard vs Walk-Forward dateFormat YYYY-MM-DD section Standard (Biased) Use ALL data (past + future): done, std1, 2020-01-01, 2021-12-31 Apply threshold to 2020-06-15 prediction: crit, std2, 2020-06-15, 1d section Walk-Forward (Unbiased) Use only data before 2020-06-15: done, wf1, 2020-01-01, 2020-06-14 Apply threshold to 2020-06-15 prediction: active, wf2, 2020-06-15, 1d
