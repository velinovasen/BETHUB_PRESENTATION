End-to-End Prediction Pipeline

Overview: This sequence diagram shows the complete flow of a single prediction request through the BETBOT API, including model routing, threshold optimization, and analytics logging. Total latency: <500ms
sequenceDiagram participant User participant API as BETBOT API participant Router as Model Router participant Model as ML Model participant Threshold as Threshold Optimizer participant DB as MongoDB participant Tracker as BETTRACKER User->>API: POST /predict {entity_id, features} API->>Router: Route by category Router->>Model: Load appropriate model Router->>DB: Fetch historical predictions DB->>Threshold: Past 180 days data Threshold->>Threshold: Optimize threshold Threshold->>Router: Optimal threshold = 0.42 Router->>Model: predict_proba(features) Model->>Router: score = 0.58 Router->>Router: 0.58 >= 0.42 → Prediction = 1 Router->>API: Response bundle API->>User: {prediction: 1, confidence: 0.58, threshold: 0.42} API->>Tracker: Log prediction for analytics Tracker->>DB: Store in performance_log Note over User,DB: Total latency < 500ms
